Game Builder Garage Switch 1984 年电影《最后的星际战士》的剧情是这样的：一名少年在一台原本不起眼的街机上取得高分，却在不知不觉中证明了自己的价值，于是被招募参加一场星际战斗。从《马力欧制造》系列到现在的 Game Builder Garage，我觉得任天堂正在做类似的事情，只不过是面向未来的游戏设计师。Game Builder Garage 为我们提供了一系列强大的工具来创作我们自己的游戏，但从我目前的成果来看，我不认为宫本茂会很快出现在我家门口邀请我加入团队。即便如此，令人印象深刻的是，在完成 Game Builder Garage 的七节互动课程后，我已经有了六款游戏的构思和制作方法。

<span>一开始我以为 Game Builder Garage 能创建的游戏类型会相当有限，就像 Super Mario Maker 虽然功能强大，但只能创建……超级马里奥关卡一样。然而，在完成所有课程（从简单的单屏“标签”游戏到更复杂的全 3D 游戏）后，我发现你能创建的游戏类型几乎没有限制。是的，你只能使用 Game Builder Garage 提供的对象和图形，但你如何构建它们以及如何与它们交互在其范围内是令人难以置信的。正如人们不断想出新的方法来制作超级马里奥制造关卡一样，我完全希望看到使用 Game Builder Garage 及其更强大的工具和功能集创建一些真正令人惊叹的游戏。</span>
<h3><span>简单的复杂性</span></h3>
<img class="aligncenter size-full wp-image-128774" src="https://sky.sfcrom.com/wp-content/uploads/2025/04/2025042706491693.webp" alt="" width="1000" height="562" />

<span>学习编程可能是一项艰巨的任务，但 Game Builder Garage 的课程非常出色，它们会引导你完成构建七种不同类型游戏所需的步骤。每节课的复杂程度都会逐渐增加，但在任何一节课中，我都没有感到迷茫或沮丧。主持人 Bob 会引导你完成每个部分的步骤，向你介绍负责每一段逻辑或子程序的 Nodon。有一些对象 Nodon 适用于简单的事物，例如盒子、圆柱体和球体。还有一些用于算术和计数等运算的 Nodon，以及布尔运算符 AND 和 NOT。没有 OR 运算符，但它的存在是通过 Nodon 的连接暗示的。</span>

<span>它让我想起了以前的计算机杂志，里面经常会刊登一些很长的 BASIC 程序。你会花上几个小时把代码输入到你当时那台破旧的家用电脑里，如果你能输入一千行 BASIC 代码让屏幕变色，你就会不知不觉地学到一些编程概念。当然，需要指出的是，Game Builder Garage 不会直接教你如何编写任何外部代码。相反，它会教你如何使用自定义游戏引擎来构建游戏。也就是说，当你安排你的 Nodon 及其流程时，你也在学习游戏的内部工作原理，而这种理解可以转化为 Game Builder Garage 界面之外的游戏设计。</span>
<h3><span>您可以构建一个简单的平台游戏或一个复杂的 3D 迷宫，其中配有移动平台、随机化和向您射苹果的巨型外星人。</span></h3>
<img class="aligncenter size-full wp-image-128773" src="https://sky.sfcrom.com/wp-content/uploads/2025/04/2025042706491568.webp" alt="" width="1000" height="562" />

<span>这些概念并不能完全转化为实际的编码，但这并不意味着 Game Builder Garage 没有一套强大的工具。恰恰相反：你可以构建一个简单的平台游戏，也可以构建一个复杂的 3D 迷宫，其中包含移动平台、复杂的随机化机制以及向你射苹果的巨型外星人。这只是意味着你不需要打开 Sublime Text 或 Notepad++ 并逐行调试代码。说实话，我有点希望有一种方法可以查看游戏创作所涉及的代码，即使它是用某种特殊的 Game Builder Garage 编程语言编写的，而不是像 C# 或 C++ 这样的常见游戏开发语言。这将有助于将游戏引擎的可视化工作流程与实际代码编写的概念联系起来。</span>

<span>Game Builder Garage 的优势在于它不需要学习编程语言的语法和特性。它让游戏创作变得更容易，但仍然并非易事。如果你期待像《超级马里奥制造》那样“所见即所得”，那你肯定会失望。Nodons 擅长以可视化的方式呈现你正在构建的游戏以及构成游戏运行的组件和部件，但它的编辑屏幕很大程度上是一种抽象。我花了一段时间才强迫自己思考游戏的各个组件在游戏画面而不是编辑画面中应该如何呈现。在这方面，这些课程帮了大忙。</span>
<h3><span>测试你的技能</span></h3>
<img class="aligncenter size-full wp-image-128772" src="https://sky.sfcrom.com/wp-content/uploads/2025/04/2025042706491569.webp" alt="" width="1000" height="562" />

<span>完成一节课（可能持续 20 分钟到 80 分钟甚至更长时间）后，Game Builder Garage 会要求你完成一系列谜题，然后才能解锁下一节课。这基本上是对你刚刚学到的概念的测验，有五个不同的谜题需要解决，这有助于在我被新的东西挤掉之前巩固这些概念。每个谜题中都嵌入了线索，主要是关于在哪里连接 Nodons 以及是否需要编辑其属性的提示。任何与正确答案无关的属性都会显示为灰色，因此你不会迷失方向寻找错误答案。</span>

我对谜题的不满在于，很多谜题一眼就能解开，而其他的……就没那么容易了。我喜欢挑战，但 10% 的谜题需要某种常数或数值范围，最终只能靠猜测。剩下的 90% 每次我卡住都是因为我忽略了某个明显的东西。任何一个花了一个小时调试实际出错的代码，结果却发现忘了关闭括号的人，都知道当你终于弄明白的时候，那种如释重负和尴尬交织的奇怪感觉。从可爱的 Nodons 到你的导师 Bob 和 Alice，它的整个呈现都非常有趣，而且个性十足。
